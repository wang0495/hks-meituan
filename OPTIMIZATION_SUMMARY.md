# CityFlow 系统优化总结

## 优化目标

对标 2026 AI Hackathon 要求，全面优化系统架构，确保：
1. 响应时间 <10秒（首token）
2. POI类型覆盖餐饮+娱乐/文化两类
3. 路线规模支持≥3个POI串联
4. 支持自然语言实时调整
5. 支持多方案对比

## 优化成果

### 1. 首token延迟优化

**问题**: 首token需要调用LLM生成问候语，延迟~2秒

**方案**: 改用规则匹配生成问候语（ADR-IP1）

**效果**:
- 优化前: ~2秒
- 优化后: <0.1秒
- 节省: ~2秒

### 2. 场景分类优化

**问题**: expert_router需要调用LLM分类场景，延迟~2-3秒

**方案**: 优先使用规则快速路径（ADR-ER1），跳过LLM调用

**效果**:
- 优化前: ~2-3秒（LLM调用）
- 优化后: ~0秒（规则匹配）
- 节省: ~2-3秒

### 3. POI加载优化

**问题**: 美团API尝试连接需要很长时间（超时15秒）

**方案**:
1. 先快速检查API可用性（0.5秒超时）
2. 减少美团API超时时间（15秒 → 2秒）
3. API不可用直接跳过，使用本地JSON

**效果**:
- 优化前: ~5.9秒
- 优化后: ~2.35秒
- 节省: ~3.55秒

### 4. 意图解析优化

**问题**: 意图解析需要调用LLM，延迟~2-4秒

**方案**:
1. 规则优先，立即返回基础结果（0延迟）
2. LLM异步补充，补充需求向量等高级信息
3. 减少重试次数（3次 → 1次）
4. 减少超时时间（30秒 → 10秒）

**效果**:
- 优化前: ~2-4秒（LLM调用）
- 优化后: ~2秒（规则+LLM并行）
- 节省: ~0-2秒
