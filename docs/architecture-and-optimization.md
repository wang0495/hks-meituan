│   {性格倾向, 兴趣图谱[tag→weight], 消费风格, 反馈历史}
├── 关键逻辑: 峰终定律(Peak-End) → 确保行程以高潮收尾
│             享乐适应 → 同类型POI不超过2个连续
│             损失厌恶 → 已预订/收藏的POI高权重保留
│             选择过载 → 推荐数量≤5个/轮
├── 实现方式: Redis Hash + 定时持久化
└── → 参考 mem0 的 LongTermMemory / 语义记忆检索

├── 峰终定律: solver生成路线后, 取前K条路线中emotion_peak最高的
├── 享乐适应: emotion.py添加 repeated_category_penalty(同类>2 → -0.3)
