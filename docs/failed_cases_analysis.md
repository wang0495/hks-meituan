## 失败案例详情

### 1. 朋友轰趴 (overall=6) ❌

**用户需求**: 5个朋友聚会，要能玩又能吃，预算人均150，下午到晚上

**系统路线**: 餐饮 -> 娱乐 -> 餐饮
- 富豪吧 [餐饮] ¥34
- 某狼人杀店 [娱乐] ¥95  
- 横琴蚝庄 [餐饮] ¥120

**LLM扣分理由**:
1. 狼人杀95元+富豪吧34元+横琴蚝庄120元=249元，超人均150预算
2. 三站距离8.5km+14.7km，地理连续性差

**质疑**: 
- 路线有3个POI且类别多样(餐饮+娱乐+餐饮)，按规则geo_continuity至少给6分
- 站间距离<25km，按规则不应扣geo_continuity分
- 预算控制确实有问题，但intent_match=7是否偏低？

- 柠檬车露 [餐饮] ¥64
- Cuppa Coffee [餐饮] ¥109

**LLM扣分理由**:
1. 柠檬车露和Cuppa Coffee凌晨2点可能已关门
2. 价格偏高，不是典型宵夜场所

- 系统没有24小时营业的POI数据，只能返回这些
- 是否应该直接拒绝而非返回错误POI？
