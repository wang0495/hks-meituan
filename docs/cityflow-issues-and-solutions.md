# CityFlow 路线规划系统 — 问题诊断与解决路径

> 最后更新: 2026-05-10
> 龙猫评分: **12/30 通过 (40%)，平均 5.7/10**

---

## 一、系统架构概览

```
用户输入 → 意图解析(DeepSeek) → LLM Planner(DeepSeek) → TSPTW求解器 → 路线输出
                                      ↓
                              推荐POI列表 + 推理理由
```

当前pipeline:
1. **意图解析** (`intent_parser.py`): DeepSeek LLM提取时间/预算/人群/偏好/硬约束
2. **LLM Planner** (`llm_planner.py`): DeepSeek根据候选POI推理推荐路线
3. **候选筛选** (`solver.py` Phase 0): 旅游相关性过滤 + 硬约束过滤 + category多样性
4. **贪心初始化** (Phase 1): 7阶段情绪曲线选POI，综合距离/情绪/疲劳/同类惩罚
5. **2-opt优化** (Phase 2): 局部搜索改进路线顺序
6. **呼吸空间** (Phase 3): 高兴奋POI间插入休息点
7. **文案生成** (`narrator.py`): LLM生成路线描述

## 三、问题深度分析

### 3.1 意图匹配是最薄弱环节 (avg 5.0/10)

意图匹配分数最低，说明系统不能准确理解用户需求。具体表现:

| 问题类型 | 频率 | 例子 |
|----------|------|------|
| **不理解隐含需求** | 高 | "喝茶听曲"→需要茶馆+曲艺，系统只推荐公园 |
| **不理解时间约束** | 高 | "3小时"→实际生成5小时路线 |
| **不理解场景语义** | 中 | "街边小吃"→推荐高端海鲜酒楼 |
| **不处理矛盾需求** | 中 | "50元住五星"→应该说明不可能 |
| **不理解情绪需求** | 中 | "私密一点"→推荐人多的公共景点 |

### 3.2 POI数据质量仍有缺陷

虽然已从999→4530个珠海POI，但:
- **深夜餐饮**: 仅22个标注24h，凌晨2点实际可选极少
- **亲子游乐**: 游乐园/海洋馆POI数量不足
- **文化休闲**: 缺茶馆、书法、曲艺等细分场景
- **夜生活**: 缺酒吧、夜店、LiveHouse
- **室内/室外标注**: 仅934个已标注（21%）

### 3.3 Solver时间约束不严格

solver的时间控制存在两个问题:
1. **结束时间弹性过大**: 曾允许超时120分钟（已修复为15分钟）
2. **总时长不被硬约束**: solver按情绪阶段选POI，不检查"总时长是否超限"
3. **LLM Planner推荐被忽略**: 推荐的POI因时间窗不符被solver拒绝

## 四、解决路径

### 路径A: 深化LLM集成（推荐优先级: ★★★）

**核心思路**: 让LLM直接决定路线，solver只做时间窗/距离优化。

#### A1. LLM直接选POI（高优先级）

当前LLM Planner只给推荐列表，solver可以忽略。改为:
- LLM Planner输出完整的POI顺序 + 时间分配
- Solver只做时间窗验证和距离优化，不改变POI选择
- 如果某个POI不可用（已关门），通知LLM重新规划

**改动量**: 中等（修改`llm_planner.py`输出格式 + `solver.py`增加"LLM指定模式"）

**预期提升**: +15-20% 通过率

#### A2. LLM意图理解增强（高优先级）

当前意图解析只提取结构化字段，不理解深层语义。改为:
- 意图解析增加`scene_description`字段，让LLM描述用户想要的体验场景
- 例如: "喝茶听曲" → `scene_description: "用户想在安静的茶馆里品茶，听传统曲艺表演"`
- Solver用`scene_description`做POI匹配，而非仅靠category和tags

**改动量**: 小（修改`intent_parser.py`的prompt + `solver.py`的匹配逻辑）

**预期提升**: +10% 通过率

#### A3. LLM路线校验（中优先级）

在solver输出后，用LLM检查路线是否合理:
- 检查时间是否超限
- 检查是否满足用户核心需求
- 检查POI质量是否达标
- 如果不合格，给出修改建议并重新求解

**改动量**: 中等（新增后处理模块）

**预期提升**: +5-10% 通过率

### 路径B: 补充POI数据（推荐优先级: ★★☆）

#### B1. 深夜/凌晨场景

用龙猫生成:
- 24h便利店（7-Eleven、全家、罗森）
- 深夜食堂/茶餐厅（营业到凌晨3-4点）
- 通宵书店/网吧/电竞馆
- 凌晨营业的药店/外卖

**目标**: 深夜可选POI从22→50+

#### B2. 亲子游乐场景

- 珠海长隆海洋王国（已有，但需要增强标签）
- 室内儿童乐园（商场内）
- 儿童科技馆/科普馆
- 亲子农场/采摘园

**目标**: 亲子相关POI从20→50+

#### B3. 文化休闲场景

- 茶馆/茶室（当前仅6个）
- 书法/国画体验馆
- 传统曲艺/粤剧场所
- 文化创意园区

**目标**: 文化休闲POI从10→30+

#### B4. 夜生活场景

- 酒吧/清吧
- LiveHouse/音乐现场
- 夜店/蹦迪场所
- 夜市/深夜小吃街

**目标**: 夜生活POI从0→15+

#### B5. 室内/室外标注补全

当前仅21%的POI有室内/室外标注。需要:
- 批量标注剩余3526个POI
- 用规则（category+tags）自动推断
- 人工校验关键POI

**目标**: 标注率从21%→80%

### 路径C: 算法优化（推荐优先级: ★★☆）

#### C1. 时间约束硬执行

当前solver按情绪阶段选POI，不检查总时长。改为:
- 在phase1循环中，每选一个POI后检查"如果继续选，总时长是否会超限"
- 如果会超限，停止添加新POI
- 将用户指定的时长转为严格的结束时间

**改动量**: 小（修改`_phase1_initialize`循环条件）

**预期提升**: +5% 通过率

#### C2. 矛盾需求处理

当前系统不处理矛盾需求（如"50元住五星酒店"）。改为:
- 意图解析阶段检测矛盾（预算 vs 需求）
- 生成"建议说明"告知用户哪些需求无法满足
- 在可满足的范围内给出最佳方案

**改动量**: 中等（修改`intent_parser.py` + `narrator.py`）

#### C3. 场景语义匹配

当前POI匹配靠category和tags，不理解语义。改为:
- 为每个POI生成"场景描述"（用LLM批量生成）
- 用语义相似度匹配用户需求和POI描述
- 例如: "喝茶听曲" ↔ "传统茶馆，提供粤曲表演"

**改动量**: 大（需要批量生成POI描述 + 语义匹配引擎）

**预期提升**: +10% 通过率

### 路径D: 评估体系优化（推荐优先级: ★☆☆）

#### D1. 分层评估

当前龙猫评分混为一体。改为分层:
- **硬性指标**: 路线非空、时间不超限、预算不超标、无黑名单POI
- **意图匹配**: 是否满足用户核心需求
- **体验质量**: POI质量、多样性、地理合理性

#### D2. 场景分类评估

按场景类型分别统计通过率:
- 基础场景（9个）: 目前100%
- 边界场景（9个）: 目前~80%
- 极限场景（12个）: 目前~25%

#### D3. 不可能需求标记

将"不可能需求"从评估中排除或单独标记，避免拉低整体分数。
